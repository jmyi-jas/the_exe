 #法院工具箱 (Court Document Toolkit)

一款基于 Python + tkinter 开发的桌面应用程序，用于自动化处理法院文书相关工作。支持 Windows 平台运行。

主要功能

1. 法院文书自动下载
   - 解析法院短信链接，自动下载电子文书
   - 支持批量处理 Excel 文件中的多条短信记录
   - 根据案号自动创建文件夹分类存储
   - 自动保存短信内容文本备份

2. 起诉状信息智能提取
   - 自动扫描文件夹中的起诉状文件（PDF/DOCX/TXT）
   - 支持普通文本 PDF 和扫描版 PDF（OCR 识别）
   - 提取要素：案号、原告名称、被告名称、第三人、一审法院、诉讼请求、事实与理由
   - 自动重命名文件夹（添加原告名称前缀）
   - 导出为 Excel 汇总表

3. 裁判文书初筛分类（判决/裁定/调解/传票）
   - 自动识别文件夹内 PDF 类型
   - 优先级：传票 > 裁定书 > 判决书 > 调解书
   - 自动添加前缀标识：【传票】、【裁定书】、【判决书】、【调解书】
   - 支持批量处理，自动跳过已标记文件夹

系统要求

- Windows 7/10/11
- Python 3.7+
- 内存：建议 4GB 以上（使用 OCR 功能时需要）

依赖安装

必需依赖：
pip install pandas requests openpyxl pdfplumber python-docx

可选依赖（用于扫描版 PDF OCR）：
pip install pymupdf easyocr numpy

注意：EasyOCR 首次运行时会自动下载模型文件（约 100MB+），需要网络连接。

使用说明

功能一：文书下载
1. 准备包含法院短信的 Excel 文件
2. 指定短信内容列名（默认：Message Body）和时间列名（默认：Date）
3. 选择输出目录
4. 点击开始，程序会按案号自动创建文件夹并下载文书

功能二：起诉状提取
1. 选择包含多个案件子文件夹的根目录
2. 程序自动扫描所有含"起诉状"字样的文件
3. 自动识别是否为扫描件，必要时启动 OCR
4. 生成"起诉状信息汇总.xlsx" 文件
5. 自动重命名文件夹为"原告名称+原案号"格式

功能三：初筛裁判书
1. 选择案件根目录（包含多个案号文件夹）
2. 程序扫描每个子文件夹内的 PDF 文件名
3. 根据文件名关键词（传票/裁定书/判决书/调解书）添加前缀
4. 优先级逻辑：传票最高，调解书最低
5. 已含前缀的文件夹会自动跳过

文件命名识别规则

- 裁定书：文件名包含"裁定书"或"本院法律文书正本_1"
- 判决书：文件名包含"判决书"
- 调解书：文件名包含"调解书"  
- 传票：文件名包含"传票"

注意事项

1. OCR 功能第一次使用需要加载模型，耗时 10-30 秒，请耐心等待
2. 确保 Excel 文件未被其他程序占用，否则无法写入结果
3. 重命名功能遇到同名文件夹会自动跳过，避免覆盖
4. 建议定期备份数据，程序操作直接修改文件名和文件夹名

技术细节

- GUI 框架：tkinter
- PDF 解析：pdfplumber（文本）+ PyMuPDF（OCR 图像转换）
- OCR 引擎：EasyOCR（支持中英文）
- Excel 处理：pandas + openpyxl
- 网络请求：requests（Session 保持）
